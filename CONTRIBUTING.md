# Contributing to UI Upgrade Playbook

Thank you for considering contributing to this framework!

## How to Contribute

### Reporting Issues
- Use GitHub Issues to report bugs or suggest features
- Include steps to reproduce
- Mention which playbook is affected

### Pull Requests
1. Fork the repo
2. Create a branch (`git checkout -b feature/amazing-feature`)
3. Make your changes
4. Test with a real project if possible
5. Commit with clear messages
6. Push and create a PR

### Writing Guidelines
- Keep language clear and actionable
- Use markdown formatting consistently
- Include examples for all prompts
- Test prompts before submitting

## Development

### Project Structure
```
ui-upgrade-playbook/
├── playbooks/          # Main playbooks (01-04)
├── design-system/      # Design tokens and guidelines
├── skills/             # Required skills setup
├── validation/         # Validation checklists
├── examples/           # Before/after examples
└── docs/               # Additional documentation
```

### Testing Changes
1. Create a test React project
2. Run through the playbook with your changes
3. Verify prompts work as expected
4. Check validation checklists pass

## Code of Conduct

- Be respectful and inclusive
- Focus on constructive feedback
- Help others learn and grow
